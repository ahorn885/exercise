2. **The orphan table *is* the gate-advancing work.** A Layer 0 table managed entirely by ad-hoc SQL outside the gate is exactly the hole the freeze can't safely close over (you can't retire the authoring loop while tables live in untracked SQL). **Two** spec-sourced orphans existed — `supplement_vocabulary` (0002) and `terrain_gap_rules` (0003) — so de-orphaning both is the gate-advancing work, discharges/parityies their Neon state, and subsumes their legacy one-shots.

## 2. What shipped (code)

- **`etl/migrations/layer0/0002_seed_supplement_vocabulary.sql`** (new) — the whole thing in **one self-contained** migration: `CREATE TABLE IF NOT EXISTS` (+ active-row partial index + two CHECK constraints) + the 25-row canonical seed (`INSERT … ON CONFLICT (supplement_id) DO NOTHING`) + a verify `DO` block, so a standalone Neon SQL-editor paste provisions the table without any separate schema step. **Shape mirrors the live one-shot DDL exactly** (bare `TIMESTAMP superseded_at`, no `etl_run_at`) — deliberately *not* normalized to the 22-table `TIMESTAMPTZ`+`etl_run_at` convention, to stay drift-free with any copy already on Neon. The `contraindications[]` already carry the canonical §B tokens, so it **subsumes both** legacy one-shots (`migrate_supplement_vocabulary.sql` base seed + `migrate_supplement_vocab_contraindication_retag_v1.sql` D-21 retag). Idempotent — provisions-or-no-ops on a fresh DB, a legacy-token prod DB, or an already-canonical prod DB.
- **Not `schema.sql`** (a first pass folded the DDL there; reverted on a CI catch). The `TestLayer0TableFamilyMap` drift guard (`tests/test_layer4_orchestrator.py`) requires every `etl_version` table in `schema.sql` to be in `_LAYER0_TABLE_FAMILY`, the `{0A,0B,0C}` cache-invalidation digest. `supplement_vocabulary` is read by Layer 2E and versioned on its own `supp_vocab.*` line — **outside** the 0A/0B/0C cone, so slice 3b deliberately kept it out of the family. Putting its DDL in `schema.sql` would force it into the family (impossible — the family is pinned to 0A/0B/0C) or break the guard. So, like the `terrain_gap_rules` precedent (created by `etl/sources/populate_terrain_gap_rules.sql`, not `schema.sql`), the table stays out of `schema.sql` and the self-contained migration is what creates it.
- **`etl/migrations/layer0/0003_seed_terrain_gap_rules.sql`** (new) — the **last** spec-sourced orphan, same self-contained shape: `CREATE TABLE IF NOT EXISTS` + the 16 canonical gap rows + the `partial`→4-band severity reclassification + a verify `DO` block, reproducing the exact live `0C-v2.0-r2` state in one transaction. **Subsumes** `etl/sources/populate_terrain_gap_rules.sql` (CREATE + 16 rows) + `etl/sources/migrate_terrain_gap_rules_severity.sql` (which also runs at app boot via `init_db._PG_MIGRATIONS`). Assembled by concatenating the two source bodies verbatim (low transcription risk over 16 text-heavy rows). Idempotent: the `DELETE`-by-version prefix rebuilds the `0C-v2.0-r2` active set on each run (D-74), `ON CONFLICT DO NOTHING` + the `partial`-only `UPDATE` no-op after the first run. **Difference from `0002`:** `terrain_gap_rules` *is* a 0C **cone** table (read by Layer 2B) already in `_LAYER0_TABLE_FAMILY` as `0C`, so the drift guard would *not* object to it being in `schema.sql` — it stays out for consistency + to keep `test_includes_out_of_schema_serving_tables` true. Already live on prod from the ad-hoc apply, so re-applying is an idempotent re-derivation, not a required deploy.
- **`etl/migrations/layer0/README.md`** — new "Tables that are not in the genesis snapshot (spec-sourced, not ETL-emitted)" section documenting the self-contained-migration absorption pattern (DDL stays *out* of `schema.sql`), with `0002` + `0003` as the worked examples.
- **`Layer0_AuthoringModel_DBSourceOfTruth_Design_v1.md`** (in-place edit, slice-3b/phase-4 precedent — no version bump): §6 item 4 records the gate **cleared** (`0001`+`0002`+`0003`) + a spec-sourced-orphan-tables sub-note; §9 checklist marks `0002`+`0003` `[x]` and the freeze gate cleared.

**The legacy one-shots** (`migrate_supplement_vocabulary.sql`, `…_retag_v1.sql`, `populate_terrain_gap_rules.sql`, `migrate_terrain_gap_rules_severity.sql`) are left in place (only doc references, no code/CI dep) — superseded by `0002`/`0003`, sweep them at the freeze.
