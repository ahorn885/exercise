## Categorization rule

**Blocker** тАФ must fix before continuing. Triggers:
- Current or imminent node would consume wrong/incomplete data because of it
- Causes silent wrong results downstream
- Blocks data the next node must produce

**Deferred** тАФ documentation only OR doesn't affect current/imminent nodes OR has a query-layer workaround. Goes to final cleanup.

**Cleanup** тАФ known spec/doc drift that has no functional impact. Pure documentation work. Lowest priority within the final cleanup batch.
